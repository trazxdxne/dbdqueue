# Dead by Daylight Queue Times & Region Locker CLI (`dbdqueue`)

A fast, zero-dependency, compiled native CLI tool for Dead by Daylight players to view real-time matchmaking queue times and manage regional matchmaking locks directly from their terminal. Compatible with both **Linux** and **Windows**.

---

## Features

- **Live Queue Times**: Fetches live queue times from `api2.deadbyqueue.com` and displays a beautifully formatted, ANSI-colored table.
- **Accurate Refresh Timestamps**: Shows the exact, true time when the queue data was last updated at the source.
- **Dynamic Sorting & Filtering**:
  - Sort by survivor times, killer times, custom priority list, or default region name.
  - Filter by game modes (Standard matchmaking, Special events, or both).
- **Region Locking**:
  - Whitelist specific AWS matchmaking regions by updating the system `hosts` file.
  - Blocks connections to unselected regions to lock you into your desired regional matchmaking server.
  - Prompts for elevated permissions (UAC on Windows / `pkexec` on Linux) on-demand only when writing DNS updates.
- **Local Settings**: Persistent config saving your preferences (mode, sorting, priority regions, locked regions):
  - Linux: `~/.config/dbdqueue/config.toml`
  - Windows: `%APPDATA%\dbdqueue\config.toml`
- **Interactive TUI Select Menus**: Easy-to-use keyboard menus (arrows + space) to select priority or locked regions, fully aligned for all terminals.

## Installation

### Linux
Install using our one-liner shell script:
```bash
curl -sSfL https://raw.githubusercontent.com/trazxdxne/dbdqueue/master/install.sh | sh
```

Alternatively, compile from source:
```bash
git clone https://github.com/trazxdxne/dbdqueue.git
cd dbdqueue
cargo build --release
sudo cp target/release/dbdqueue /usr/local/bin/
```

### Windows
1. Download the latest compiled Windows binary `dbdqueue-windows-x64.exe` from our [GitHub Releases](https://github.com/trazxdxne/dbdqueue/releases) page.
2. Rename it to `dbdqueue.exe`.
3. Add the folder containing the executable to your user `PATH` environment variable so you can run it from any PowerShell or Command Prompt.

Alternatively, compile from source (requires Rust toolchain):
```powershell
git clone https://github.com/trazxdxne/dbdqueue.git
cd dbdqueue
cargo build --release
# The compiled executable will be in: target\release\dbdqueue.exe
```

## Usage

### 1. View Live Queue Times
Simply run `dbdqueue` to view the live queues table:
```bash
dbdqueue
```

### 2. Configure Sort Preferences
Sort by survivor or killer queue times, priority regions, or reset to default:
```bash
dbdqueue --sort survivor
dbdqueue --sort killer
dbdqueue --sort priority
dbdqueue --sort default
```
*Note: Your sorting choice is saved in the configuration file and will be used as the default for subsequent runs.*

### 3. Filter by Mode
Filter between standard matchmaking and event matchmaking:
```bash
dbdqueue --mode standard
dbdqueue --mode event
dbdqueue --mode both
```

### 4. Manage Custom Priority Regions
To place your preferred regions at the top of the table:
```bash
dbdqueue --priority "Frankfurt, Dublin, Virginia"
```
Or run with empty priority to open the **interactive setup menu**:
```bash
dbdqueue --priority
```

### 5. Region Locking
Lock your client to one or more whitelisted matchmaking regions (e.g., only Dublin and Frankfurt):
```bash
dbdqueue lock Dublin Frankfurt
```
Or open the **interactive lock menu** to choose via checkboxes:
```bash
dbdqueue lock
```
To unlock and restore your hosts file back to normal:
```bash
dbdqueue unlock
```

## How Region Locking Works Under the Hood

Dead by Daylight matchmaking routes game client connections to specific AWS region endpoints.
When you lock to specific regions, `dbdqueue` maps region names (e.g. `Frankfurt`) to AWS codes (e.g. `eu-central-1`). It updates your system `hosts` file to route ping and game endpoints for all *non-whitelisted* AWS regions to loopback IP addresses (`0.0.0.0` and `::1`). This forces the game client to only communicate with your whitelisted region(s).

The hosts path resolved is:
- **Linux**: `/etc/hosts`
- **Windows**: `C:\Windows\System32\drivers\etc\hosts`

The tool wraps edits in a delimited block:
```hosts
# --- DBD REGION CHANGER START ---
# Do not edit manually. Generated by dbdqueue CLI
0.0.0.0 gamelift-ping.us-east-1.api.aws
::1 gamelift-ping.us-east-1.api.aws
...
# --- DBD REGION CHANGER END ---
```

When you run `dbdqueue unlock`, this entire block is cleanly removed.
To avoid requiring administrator privileges for normal time checks, the tool prompts for elevated permission on-demand only when writing to the hosts file:
- On **Linux**, it pipes updates via `pkexec tee`.
- On **Windows**, it triggers a UAC pop-up request to copy the temporary configuration file over.

## Technical Specifications

- **Language**: Rust
- **Dependencies**: 
  - `clap`: Command-line parser
  - `ureq`: Lightweight HTTP client
  - `serde_json` & `serde`: Structured JSON parsing from API
  - `crossterm`: Terminal raw mode and input handling (fully cross-platform)
  - `toml`: Config storage formatting
  - `chrono`: Displaying local/api timestamps
  - `regex`: ANSI removal and time matching
- **Platform Support**: Linux and Windows.
