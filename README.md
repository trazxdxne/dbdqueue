# Dead by Daylight Queue Times & Region Locker CLI (`dbdqueue`)

A fast, zero-dependency, compiled native CLI tool for Dead by Daylight players to view real-time matchmaking queue times and manage regional matchmaking locks directly from their terminal.

---

## Features

- **Live Queue Times**: Fetches live queue times from `api.deadbyqueue.com` and displays a beautifully formatted, ANSI-colored table.
- **Dynamic Sorting & Filtering**:
  - Sort by survivor times, killer times, custom priority list, or default region name.
  - Filter by game modes (Standard matchmaking, Special events, or both).
- **Region Locking**:
  - Whitelist specific AWS matchmaking regions by updating `/etc/hosts`.
  - Blocks connections to unselected regions to lock you into your desired regional matchmaking server.
  - Uses `pkexec` to elevate privileges safely on-demand only when writing DNS updates.
- **Local Settings**: Persistent config at `~/.config/dbdqueue/config.toml` saving your preferences (mode, sorting, priority regions, locked regions).
- **Interactive TUI Select Menus**: Easy-to-use keyboard menus (arrows + space) to select priority or locked regions.

## Installation

Install using our one-liner shell script:

```bash
curl -sSfL https://raw.githubusercontent.com/trazxdxne/dbdqueue/master/install.sh | sh
```

*(Note: Replace `trazxdxne/dbdqueue` in `install.sh` with the correct git repository path if self-hosting.)*

Alternatively, you can compile from source:

```bash
git clone https://github.com/trazxdxne/dbdqueue.git
cd dbdqueue
cargo build --release
sudo cp target/release/dbdqueue /usr/local/bin/
```

## Usage

### 1. View Live Queue Times
Simply run `dbdqueue` to view the live queues table:
```bash
dbdqueue
```

### 2. Configure Sort Preferences
Sort by survivor or killer queue times, priority regions, or reset to default:
```bash
dbdqueue --sort survivor
dbdqueue --sort killer
dbdqueue --sort priority
dbdqueue --sort default
```
*Note: Your sorting choice is saved in the configuration file and will be used as the default for subsequent runs.*

### 3. Filter by Mode
Filter between standard matchmaking and event matchmaking:
```bash
dbdqueue --mode standard
dbdqueue --mode event
dbdqueue --mode both
```

### 4. Manage Custom Priority Regions
To place your preferred regions at the top of the table:
```bash
dbdqueue --priority "Frankfurt, Dublin, Virginia"
```
Or run with empty priority to open the **interactive setup menu**:
```bash
dbdqueue --priority
```

### 5. Region Locking
Lock your client to one or more whitelisted matchmaking regions (e.g., only Dublin and Frankfurt):
```bash
dbdqueue lock Dublin Frankfurt
```
Or open the **interactive lock menu** to choose via checkboxes:
```bash
dbdqueue lock
```
To unlock and restore your hosts file back to normal:
```bash
dbdqueue unlock
```

## How Region Locking Works Under the Hood

Dead by Daylight matchmaking routes game client connections to specific AWS region endpoints.
When you lock to specific regions, `dbdqueue` maps region names (e.g. `Frankfurt`) to AWS codes (e.g. `eu-central-1`). It updates your `/etc/hosts` file to route ping and game endpoints for all *non-whitelisted* AWS regions to loopback IP addresses (`0.0.0.0` and `::1`). This forces the game client to only communicate with your whitelisted region(s).

The tool wraps the `/etc/hosts` edits in a delimited block:
```hosts
# --- DBD REGION CHANGER START ---
# Do not edit manually. Generated by dbdqueue CLI
0.0.0.0 gamelift-ping.us-east-1.api.aws
::1 gamelift-ping.us-east-1.api.aws
...
# --- DBD REGION CHANGER END ---
```

When you run `dbdqueue unlock`, this entire block is cleanly removed, restoring `/etc/hosts` to its original state.
To avoid requiring root access for the entire execution, the utility invokes `pkexec tee` to edit `/etc/hosts` with elevated privileges on-demand, prompting for password authorization only when a modification is actually needed.

## Technical Specifications

- **Language**: Rust
- **Dependencies**: 
  - `clap`: Command-line parser
  - `ureq`: Lightweight HTTP client
  - `crossterm`: Terminal raw mode and input handling
  - `toml`: Config storage formatting
  - `chrono`: Displaying last-update timestamp
  - `regex`: ANSI removal and string processing
- **Platform Support**: Linux (required for host-file updates and `pkexec` wrapper).
